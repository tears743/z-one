## 定价公式

```
项目报价 = 工时 × 时薪 × 复杂度系数 + 加急费

时薪参考：
- 初级：50-80元/小时
- 中级：100-150元/小时  
- 高级：200-300元/小时
- 专家：300-500元/小时

复杂度系数：
- 简单：1.0
- 中等：1.3
- 复杂：1.8
- 高度定制：2.5+
```

## 定价策略建议

### 1. 新手起步价
- 前3单：市场价的 **60-70%**（积累评价）
- 4-10单：市场价的 **80-90%**（建立口碑）
- 10单后：市场价或溢价

### 2. 议价空间
- 永远留出 **10-20%** 的议价空间
- 客户砍价时，可以增加服务而不是降价
- 例："如果今天确认，我免费加上XXX功能"

### 3. 加急费标准
- 正常工期的 **50%** 时间要求 → 加急费 **30%**
- 正常工期的 **30%** 时间要求 → 加急费 **50%**
- 正常工期的 **20%** 时间要求 → 加急费 **100%**

### 4. 维护费
- 免费维护期：**7-30天**（根据项目大小）
- 后续维护：**项目金额的10-20%/年**
- 或按次收费：**200-500元/次**

## 报价话术模板

### 初次报价
```
根据您的需求，这个项目我评估下来需要X天完成，
报价是XXXX元（包含X次免费修改）。

交付内容：
✓ 源代码
✓ 部署文档
✓ 30天免费维护

如需加急（X天内交付），加急费为XXX元。
```

### 客户嫌贵时
```
我理解您对预算的考虑。

我们可以这样调整：
方案A：保留核心功能，简化XXX，价格XXXX元
方案B：分期交付，先做MVP版本XXXX元
方案C：如果您能接受X周的排期，可以优惠XX%

您觉得哪种更合适？
```

### 涨价通知（老客户）
```
感谢您一直以来的信任！

由于我近期升级了技术栈/交付流程，
从X月X日起，我的服务价格将调整为XXXX。

在X月X日前确认的项目，仍可享受原价格。
期待继续为您服务！
```

## 定价检查清单

报价前确认：
- [ ] 充分理解需求（避免后期变更）
- [ ] 评估实际工时（乘以1.5的安全系数）
- [ ] 确认客户预算范围
- [ ] 明确交付标准和验收流程
- [ ] 确定付款方式（建议3-4-3或5-5）
