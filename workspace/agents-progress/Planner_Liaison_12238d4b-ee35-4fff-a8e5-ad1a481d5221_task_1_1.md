## 澄清问题清单（最少必要）

1) **主题与受众**
- 这份 Markdown 正文的主题是什么？面向谁（新手/专业/内部/对外）？

2) **目标与交付边界**
- 你希望读者看完做什么（了解/决策/下单/复盘）？
- 正文需要覆盖的范围：只写正文，是否包含标题、目录、结论、附录？

3) **内容骨架（最小结构确认）**
- 是否采用固定结构：`摘要 → 正文要点（分小节）→ 结论/行动建议 → 附录（可选）`？
- 你是否已有必写要点/关键词清单？（有则直接贴）

4) **素材与事实来源（可空但需确认）**
- 有无参考资料/链接/数据表/截图？请提供（没有也请明确“无”）。
- 是否需要“引用格式”（脚注/括号链接/参考文献列表）？

5) **数据来源 / 时间戳 / 风险提示（必须确认）**
- 是否在文末或文首包含：
  - 数据来源（Source/References）？（是/否）
  - 时间戳（截至何时，含时区）？（是/否；如是：日期+时区）
  - 风险提示/免责声明？（是/否；如是：偏“投资风险/合规/一般性信息”哪种）

6) **价格锚点策略（必须二选一）**
- 全文涉及“价格/涨跌/回撤/目标”等表述时，采用哪种锚点？
  - **P0：固定基准价**（你提供一个 P0 数值+币种/单位+基准时间）
  - **最新价：写作时最新价**（你提供最新价数值+获取时间；或授权我用你提供的数据源里最新一条）
- 若两者都不方便：是否允许**不写具体价格，只写区间/相对变化**？

7) **风格与长度**
- 语气：中性客观 / 推广营销 / 研究报告 / 口语科普？
- 目标长度：大约多少字或多少段/小节？

8) **格式偏好（影响落盘）**
- 是否需要：表格/要点清单/代码块/公式/引用块？
- 标题层级：最多到 `##` 还是需要 `###/####`？

## 接收分段规则（你可以分多次发，我会按规则拼成最终 Markdown）

1) **每段用以下标记开头**（复制即可）  
- `【段落1/总n】`、`【段落2/总n】` …  
或如果不知道总数：`【段落1】`、`【段落2】` …

2) **每段只发“正文内容”**  
- 不要混入额外说明；若必须说明，用 `备注：` 单独一行。

3) **引用/数据**  
- 链接直接贴原始 URL；数据表可用 Markdown 表格或 CSV 粘贴。

4) **默认合并规则**  
- 我会按段落序号顺序拼接；如你发来“替换段落”，用：`【替换 段落k】`。

5) **结束信号**  
- 发送：`【完】` 表示素材提交完毕，我将输出完整 Markdown 正文（并按你选择包含/不包含数据来源、时间戳、风险提示与价格锚点策略）。

---

你先回答第 **5**、**6** 两项（是否含来源/时间戳/风险提示 + 价格锚点选 P0 或最新价），其余项可随后分段补充。
