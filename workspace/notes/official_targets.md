# 官方入口清单（用于快速判真伪/版本）

目标：在 UI 检索时快速识别 **“节目单/节目列表”** 最可能出现的官方页面与标题特征。

优先级（证据力从强到弱）：
1. **央视网（CCTV.com）春晚官方专题页/活动页**（常含：直播入口 + 回看 + 节目单/节目单更新）
2. **央视新闻（news.cctv.com / 央视新闻客户端内容外链）通稿/快讯**（常发布：节目单、主持人阵容、直播时间、回看引导）
3. **央视频（yangshipin.cn / 央视频 App）节目页/合集页/直播回放页**（常含：节目分段/节目切条，但也可能只给视频列表而非文字节目单）

---

## A. CCTV.com（央视网）— 春晚专题/节目单承载主阵地

### A1. 官方专题首页（最强入口）
**页面类型**：年度春晚“官方专题页/活动页/主会场页”
- 常见 URL 形态（示例特征，非固定）：
  - `https://...cctv.com/.../chunwan/`（路径包含 `chunwan` / `春晚` / 年份）
  - 专题二级域/栏目聚合：`tv.cctv.com`、`ent.cctv.com`、`cctv.com/special/` 等
- **标题特征（高可信）**：
  - `2026年中央广播电视总台春节联欢晚会`（或简称 `2026春晚`）
  - `中央广播电视总台2026年春节联欢晚会`
  - `春节联欢晚会 官方网站/专题`（页面 title/导航会出现“专题”“活动”“主会场”）
- **页面结构特征（贴近原始结构）**：
  - 顶部导航：`首页 | 新闻 | 视频 | 直播 | 节目单 | 精彩节目 | 花絮 | 幕后 | 互动`
  - 核心模块：`直播中/回看` + `节目单/节目单更新`（锚点或单独 tab）

**节目单出现形式**：
- Tab/锚点：`节目单`、`节目单来了`、`完整节目单`、`节目单（以播出为准）`
- 列表结构常见字段：`节目序号` / `节目名称` / `表演者` / `类型（歌舞/小品/相声等）` / `播出顺序`

### A2. 专题内“节目单”独立页（直接命中）
**页面类型**：专题二级页面（通常标题直指节目单）
- **标题特征**：
  - `（完整/正式/最终）节目单`、`节目单公布`、`节目单出炉`、`节目单更新`、`节目单一览`
  - `节目单（直播版/播出版/回看版）`
- **判真伪要点**：
  - 必须能从 A1 专题页导航跳转（或在同专题域/路径下）
  - 文内常带 `央视网` 署名/责任编辑/来源字段（不一定每页都有）

### A3. tv.cctv.com（央视网视频）— 春晚节目合集/回看与“节目列表”
**页面类型**：视频节目页、栏目页、合集页、回看页
- 常见 URL 形态：
  - `https://tv.cctv.com/...`（节目视频、合集、栏目）
- **标题特征**：
  - `《2026年春节联欢晚会》` + `直播`/`完整版`/`回放`/`精编版`
  - `春晚精彩节目`、`春晚节目集锦`、`春晚回放`
- **节目单/列表出现形式**：
  - 更常见为：`视频列表/分P/节目切条`（每个节目一个视频）
  - 文本“节目单”未必完整，但视频列表可反推节目顺序

## B. 央视新闻（news.cctv.com / 央视新闻体系）— 通稿式“节目单公布/更新”

### B1. 通稿/快讯/图文稿（高可信）
**页面类型**：新闻稿（正文直接列出节目单，或发布“节目单更新公告”）
- 入口域名与形态：
  - `https://news.cctv.com/`（央视新闻网站）
  - 央视新闻客户端内容外链（落地也常在 `cctv.com` 体系）
- **标题特征**（极常见模板）：
  - `节目单公布！2026年春晚节目单来了`
  - `2026年央视春晚节目单正式发布/出炉`
  - `总台春晚节目单（完整版）`
  - `春晚节目单更新`、`春晚节目单公布（以播出为准）`
  - `主持人阵容公布`（有时与节目单同稿/同专题）

**正文结构特征**：
- 开头 1-2 段说明：直播时间/平台、节目亮点
- 中段用列表/分段给出节目单：
  - `序号 + 节目名称 + 表演者`（常用顿号/括号）
  - 或表格化结构
- 末尾含 `责任编辑`、`来源：央视新闻`/`央视网` 等字段

### B2. “辟谣/更正/以播出为准”类公告（用于版本判定）
**页面类型**：澄清稿/提示稿
- **标题特征**：
  - `网传节目单不实`、`所谓“节目单”系谣言`、`以最终播出为准`
  - `节目单仍在调整中`、`节目顺序以直播为准`
- **用途**：
  - 给 UI 做“版本标签”：`官方非最终/辟谣`，避免把网传联排单当最终版

## C. 央视频（yangshipin.cn）— 直播/回放页与节目分段列表

### C1. 春晚直播页/回放页（强入口，但文字节目单不一定完整）
**页面类型**：央视频活动页（直播）与回放落地页
- URL 域名：`yangshipin.cn`
- **标题特征**：
  - `2026年春晚直播`、`春晚直播间`、`中央广播电视总台春节联欢晚会 直播`
  - `2026春晚 回放/完整版/精彩回看`
- **页面结构特征**：
  - `直播播放器` + `预约/提醒` + `聊天室/互动`
  - 回放后常出现：`选集/切条/节目`列表（每段一个视频）

### C2. “合集/专辑/选集”页（可作为节目列表替代证据）
**页面类型**：节目合集（把每个节目切条成列表）
- **标题特征**：
  - `春晚节目合集`、`春晚精彩节目（合集）`、`春晚节目单（视频版）`
- **判别要点**：
  - 若页面只给视频标题列表：仍可作为“节目列表”证据，但需标注 `央视频切条列表（非文字节目单）`

## D. UI 快速判真伪：标题/域名/结构三要素

### D1. 域名白名单（强建议）
- `cctv.com`（含 `tv.cctv.com`、`news.cctv.com` 等）
- `yangshipin.cn`

### D2. 高可信标题关键词（命中即优先抓取）
- `中央广播电视总台` + `春节联欢晚会`
- `2026年` + `节目单` / `完整节目单` / `正式节目单` / `最终节目单`
- `节目单公布/出炉/来了/一览`
- `回放/完整版`（配合节目列表/选集）

### D3. 低可信/需谨慎标题特征（常见“网传联排单”外观）
- `网传`、`曝光`、`疑似`、`内部`、`流出`、`独家`
- `第X次联排节目单`（除非页面明确来自央视官方账号/稿件）
- 无明确来源字段、无央视域名或无法回链到央视

---

## E. 建议 UI 抽取的“节目单”结构字段（便于统一展示）
- 年份/届次：`2026`
- 来源平台：`CCTV.com / 央视新闻 / 央视频`
- 页面类型：`专题页/新闻稿/直播页/回放页/合集页`
- 节目条目字段（尽量贴近原始）：
  - `序号（如有）`
  - `节目名称`
  - `表演者/演员阵容`
  - `节目类型（如有）`
  - `播出顺序/时间（如有）`
- 版本标签：`最终播出版 / 官方预告版（以播出为准） / 辟谣澄清 / 切条列表（视频版）`
